# Next Fun Club v1.2 更新公告草稿

状态：进行中，尚未发布到 `main`

v1.2 重点调整活动与组队的产品模型：外部公开活动不再被当作“可报名队伍”，而是作为活动信息位展示；用户可以基于活动信息发起自己的组队。

## 主要亮点

1. 活动信息和组队活动开始拆分。
2. 活动发现页仍保持原有卡片体验，但语义更清楚。
3. 公开活动可以作为信息位查看，也可以用来发起组队。
4. 组队大厅入口更清晰，适合查看正在约人的队伍。

## 用户可感知更新

1. 从公开来源导入的活动会更像“活动信息”，不再强行显示报名人数。
2. 用户可以在活动信息页查看已有组队，也可以基于该活动发起自己的组队。
3. 已经存在的组队不会再继续派生新的组队，避免层级混乱。
4. 活动发现页保留原有卡片排布，但按钮和标签会区分活动信息与组队。
5. 组队大厅用于集中查看正在约人的队伍。
6. 旧导入活动会兼容新的活动信息位逻辑，减少 `0/99 人` 这类误导展示。

## 发布前确认

1. v1.2 功能完成并合并到 `main` 后，再把这个草稿同步到 `/updates` 页面。
2. 未发布前不要在公开更新列表中展示 v1.2。
