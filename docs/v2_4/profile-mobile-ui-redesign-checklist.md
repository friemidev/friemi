# Profile 主页移动端 UI 重做 Checklist

> 目标：把当前偏“资料管理 + 组局列表”的个人主页，调整成更像真实社交产品的 Profile 页面。自己的主页、组局列表、好友列表、他人主页需要分开设计，避免一个组件同时承担太多场景。

## 0. 本阶段实施边界

本阶段只做 Profile 的移动端框架重做和已有功能接入，不新增复杂业务闭环。

- [x] 已有功能可以接入：头像、昵称、好友码、简介、发起的组局、参与的组局、收藏的组局、好友列表、加好友 / 发消息、公开他人主页。
- [x] 暂无后端或规则未定的功能只保留入口按钮，不实现业务逻辑。
- [ ] 未完成入口统一走轻提示或“敬请期待”，不能跳到空白页。
- [x] 不为本次 UI 框架新增数据库表或复杂接口。
- [ ] 不做访客记录、礼物墙、背包、充值、商城、成就、魅力值计算、信用值规则、粉丝 / 关注闭环。
- [ ] 他人主页的时间线优先接已有公开足迹和公开组局；如果数据暂不稳定，可以先保留页面结构和空状态。

## 1. 本次需求来源

当前页面问题：

- 自己的个人主页仍然像后台资料页，信息密度高，缺少头像、昵称、城市、信用值、入口宫格等明确层级。
- 组局、参与、收藏列表和好友列表都挤在一个页面里，切换后移动端阅读压力较大。
- 目前“足迹里的主页”只是临时承接个人主页能力，不适合作为最终 Profile 体验。
- 他人主页与自己的主页共用同一套布局，缺少社交关系、魅力值、最近动态、加好友 / 发消息等真实用户视角。

## 6. 他人 Profile 页面

对应截图四。

### 顶部与用户信息

- [x] 顶部左侧返回。
- [x] 顶部右侧分享和更多按钮。
- [x] 展示头像、昵称、账号名 / 好友码、城市。
- [ ] 展示在线状态或最近活跃状态时，不能泄露隐私；没有数据则不展示。
- [ ] 显示标签：共同好友、关系状态、兴趣标签等；没有数据时隐藏。

### 社交操作

- [x] 如果不是好友：显示 `加好友`。
- [x] 如果好友请求已发送：显示 `已申请`。
- [x] 如果已经是好友：显示 `发消息`，并保留更多操作里的删除好友。
- [ ] 操作按钮要贴近截图四：小号圆角胶囊按钮，不使用大面积主按钮。

### 信用 / 魅力值

- [x] 信用 / 魅力值本阶段不做真实计算。
- [x] 如果需要视觉上保留，可只显示入口或静态占位，不显示会误导用户的具体分数。
- [ ] 不要让分值看起来像金融信用评分；文案要偏社交娱乐。
- [ ] 礼物墙入口可以保留为未来能力。

### 时间线内容

- [ ] 以日期分组展示该用户最近公开动态 / 公开参与记录。
- [x] 可先接入公开足迹和公开组局。
- [ ] 图片、视频或组局卡片采用小缩略图，不做长列表大卡堆叠。
- [ ] 非好友只显示公开内容；好友可见内容必须按权限判断。
- [x] 未登录用户可查看公开主页，但加好友、发消息需要登录。

- [x] `getProfileDashboard`：自己的发起、参与、收藏、好友、狼人杀统计等。
- [x] `getPublicProfileDashboard`：他人公开资料与关系状态。
- [ ] `ProfileActivitySections`：当前组局列表逻辑可拆到独立页面。
- [ ] `ProfileOverviewPanel`：当前好友面板逻辑可拆到独立页面。
- [ ] `ProfileSocialActions`：他人主页关系按钮可复用，但需要换成新 UI。

需要确认或补充的数据：

- [x] 足迹数量：Profile 顶部的 `Moments / 足迹` 统计是否已有稳定查询。
- [ ] 粉丝 / 关注：当前如果没有真实 follow 模型，第一版只展示好友，不做假数据。
- [ ] 访客记录：本阶段只保留入口，不做真实记录。
- [ ] 魅力值 / 信用值：本阶段只保留入口或静态占位，不做计算规则。
- [ ] 他人时间线：需要统一公开足迹、公开组局、公开参与记录的权限规则。

## 8. 移动端 UI 规范

- [ ] 只优先重做移动端，桌面端暂时保持可用，不做大规模重排。
- [ ] 页面宽度遵循现有移动端容器，避免左右溢出和 WebView 横向滚动。
- [ ] 不使用当前旧 Profile 的大量嵌套框框。
- [ ] 头像、统计、宫格、列表之间留白要更像社交 App，而不是管理后台。
- [ ] 文字要短，避免说明书式文案。
- [ ] 卡片圆角、阴影、边框延续 v2_4 移动端轻量风格。
- [ ] 底部导航栏保持现有样式，不在本任务里重做。

## 9. 权限与隐私

- [ ] 自己主页可以看到全部发起、参与、收藏。
- [ ] 他人主页默认只展示公开内容。
- [ ] 好友可见内容只在双方为好友时展示。
- [ ] 未登录用户访问他人主页时，不应触发强制登录，除非点击加好友 / 发消息。
- [ ] 好友码可以在自己主页展示；他人主页是否展示好友码需要确认。
- [ ] 访客记录如未完成隐私设计，不要上线真实记录。
