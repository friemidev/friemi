# v2.4 通知与竖屏清理 Checklist

## 背景

本次分支处理两个问题：

- 动态和私聊已经有自己的页面入口，不应该再重复写入通知中心，避免用户同时看到“消息”和“通知”两套提醒。
- 当前移动端页面只按竖屏设计，横屏会挤压底部导航、弹窗、卡片和输入框，所以需要尽量锁定竖屏，并保留横屏兜底提示。

## 验收清单

- [ ] iOS App 内旋转手机后仍保持竖屏

- [x] TypeScript 检查通过

## 说明

普通手机浏览器不一定允许网页主动强制锁定方向，尤其是非 PWA、非全屏场景。因此 Web 侧采用三层策略：manifest 声明竖屏、客户端尝试锁定竖屏、失败时显示横屏提示遮罩。App 侧使用原生配置锁定竖屏。
