## 设计方向

这组卡片不继续沿用旧的橙棕纸片感，而是切到 v2.1 的主品牌体系：

- 活动卡片：轻量发现票券。更像“我发现了一个值得去看的活动”，操作少、信息清楚、阅读轻。
- 组局卡片：小聚会邀请牌。更强调人、报名状态、成员头像和行动入口，视觉上更有活力。
- 共同气质：奶油纸面、品牌绿边界、深蓝文字、珊瑚行动色、浅沙金分隔。

目标不是把卡片做得花，而是在 390px 双列和 1280px 桌面网格里都保持清爽、有识别度、略微俏皮。

## 活动卡片

### 桌面端

- 宽度建议：`250px-300px`，1280px 以 4 列为优先，超宽屏再考虑 5 列。
- 图片比例：`16:10`，顶部圆角比卡片小 2px，图片 hover 轻微放大。
- 类型 badge：左上角小票签，使用 `event blue`，文案短，例如“活动 / 展览 / 音乐”。
- 状态角标：右下贴近图片底部，淡底深字，不抢标题。
- 收藏按钮：右上角白色圆钮，保持可点击但不要遮住状态。
- 内容：标题 2 行，日期和地点各 1 行，描述不在列表卡片里展开。
- 操作：活动资讯卡片不放大 CTA，整卡可点；需要入口时用低调“查看详情”文本链。

### 移动端

- 双列卡片宽度按容器自适应，最小可读宽度不低于 `150px`。
- 图片高度建议 `96px-112px`，标题最多 2 行。
- badge 文案优先短标签，英文/法语使用短词，避免按钮内长句。
- 日期可压缩成一行，地点只保留城市或短地点；完整地址进详情页。

## 组局卡片

### 桌面端

- 组局卡片比活动卡片多一个“人”的层级：报名人数、头像叠放、发起人/我发起状态。
- 顶部类型 badge 使用珊瑚底 + 品牌绿小点，和活动卡片拉开语义。
- 卡片边框可略偏珊瑚，但底色仍保持奶油，避免整块变红。
- 报名人数与成员头像在同一行，形成社交信号。
- 操作入口使用一个合并动作区：
  - 默认是短胶囊或小拨片，不显示长文案。
  - hover 左侧时展开“报名加入”。
  - hover 右侧时展开“再来一局”。
  - hover 面必须覆盖整个按钮底，不能出现动画遮不住的断层。

### 移动端

- 移动端不使用 hover 动画。
- 卡片内只保留一个轻量动作按钮，点击后打开 bottom sheet。
- bottom sheet 使用 viewport 级 portal，不挂在卡片内部，避免被双列卡片、底部导航或滚动容器裁切。
- bottom sheet 顶部显示活动标题，下面两项操作：报名加入、再来一局。
- bottom sheet 入场使用轻量遮罩淡入 + 面板上滑，出场淡出下滑后再卸载；时长控制在 `180ms-220ms`，不影响连续点击。
- 卡片内动作按钮文案要短，中文可用“操作”，英文/法语可用 `Actions` / `Options`，图标承担语义。
- 头像叠放最多显示 3 个，超出显示 `+N`，尺寸 `18px-22px`。

## 细节规则

- 卡片圆角：桌面 `20px-22px`，移动端 `16px-18px`。
- 卡片边框：默认 `1px solid #d8e7cf`，组局 active/hover 可切 `#f3b1a7`。
- 阴影：只用低透明度绿影或珊瑚影，不使用重黑阴影。
- hover：桌面卡片上浮 `-3px`，图片放大 `1.03`，badge 轻微位移即可。
- active：移动端点击缩放 `0.985`，不要改变布局尺寸。
- 标题：桌面 18px 左右，移动端 14px-15px；不要在小卡片里使用 hero 字号。
- 信息图标：使用 lucide 图标，尺寸桌面 `15px-16px`，移动端 `13px-14px`。
- 文案截断：标题 2 行，地点 1 行，长按钮文案不直接放入移动端卡片。

## 后续落地建议

1. 先抽共享的 `CardShell` / `CardMedia` / `CardBadge` / `CardMetaRow` 样式，避免活动和组局各写一套。
2. 活动卡片先落地移动端双列，因为最容易暴露图片比例、标题截断和收藏按钮遮挡问题。
3. 组局卡片再落地动作区，桌面实现 hover split，移动端实现 bottom sheet。
4. 最后统一搜索页、个人主页和首页 carousel，避免同一种活动卡在不同页面长得不同。
